## **뉴스 요약 자동화 워크플로우 단계 설명** written by 김태경

<img width="2461" height="1204" alt="image" src="https://github.com/user-attachments/assets/b253a57d-547b-4101-8be9-8e469c50e598" />

**1. Schedule Trigger**

    워크플로우가 정해진 시간에 자동 실행되도록 설정하는 시작 노드이다. 
    현재 워크플로우는 매일 오전 9 시에 실행되도록 구성되어 있으며,
    이 시간이 되면 전체 뉴스 수집·필터링·요약·저장 과정이 자동으로 시작된다. 
    사용자가 직접 실행 버튼을 누르지 않아도 정해진 시간에 뉴스 자동화 시스템이 작동한다. 
    
**2. Set Topic**

    수집할 뉴스 주제를 설정한다. 
    (예: 경제, IT, 국제, 정치, 생활 등.) 
    Set Topic 노드는 독립적으로 실행되지 않으며, 
    Schedule Trigger 가 실행될 때 함께 실행된다. 
    즉, 현재 설정된 주제는 매일 오전 9 시 Schedule Trigger 가 동작하는 시점에 적용되며, 
    이후 RSS 수집 및 뉴스 필터링 과정 전체에 반영된다. 
    
    예를 들어 Set Topic 값이 “경제”로 설정되어 있다면, 
    오전 9시에 워크플로우가 실행될 때 경제 관련 기사만 선별하여 분석 및 요약을 수행한다. 
    
**3. RSS 3 개 수집**

    연합뉴스, 한국경제, 파이낸셜뉴스의 RSS 피드에서 최신 기사 데이터를 가져오는 단계이다.
    각 RSS 노드는 해당 언론사의 기사 목록을 불러오며, 
    기사 제목, 원문 링크, 기사 설명, 발행일시 등의 원본 데이터를 수집한다. 
    여러 언론사의 데이터를 함께 활용함으로써 특정 매체에만 의존하지 않고 다양한 출처의 뉴스를 확보한다. 

**4. Normalize 3 개**

    각 언론사 RSS 데이터의 구조를 공통 형식으로 정리하는 단계이다. 
    언론사마다 RSS 필드명이 다를 수 있기 때문에,
    제목은 title, 링크는 link, 설명은 description, 발행일시는 publishedAt, 언론사는 source로 통일한다. 
    이 과정을 통해 이후 병합, 중복 제거, 필터링, AI 분석 단계에서 데이터를 일관되게 처리할 수 있다. 
    
**5. Merge Sources**

    세 개 언론사에서 각각 수집·정리된 기사 데이터를 하나의 목록으로  합치는 단계이다. 
    연합뉴스, 한국경제, 파이낸셜뉴스에서 나온 기사들이  하나의 통합 기사 목록으로 합쳐지며, 
    이후 단계에서는 이 통합 목록을 기준으로 중복 제거와 주제 필터링을 수행한다. 
    여러 소스를 하나로 묶는 허브 역할을 한다. 

**6. Deduplicate News**

    동일하거나 유사한 기사가 중복으로 처리되지 않도록 제거하는 단계이다. 
    주로 원문 링크와 기사 제목을 기준으로 중복 여부를 판단한다. 
    같은 기사가 여러 번 저장되거나 AI 선정 과정에서 반복 노출되는 것을 방지하여, 최종 결과의 품질을 높인다. 
    
**7. Balance Sources**

    특정 언론사의 기사만 과도하게 많이 포함되지 않도록 기사 목록을 균형 있게 섞는 단계이다.
    RSS 수집량이 많은 언론사의 기사만 상위에 몰릴 경우 AI 가 한쪽 매체 기사만 선택할 수 있기 때문에,
    언론사별 기사가 고르게 반영되도록 순서를 조정한다. 
    이를 통해 결과가 특정 출처에 편중되는 문제를 줄인다. 
    
**8. Topic Filter**

    Set Topic 에서 지정한 주제와 관련된 기사만 선별하는 단계이다.
    예를 들어 Set Topic 이 “경제”라면 주식, 증시, 금리, 환율, 투자, 반도체, 부동산 등 
    경제 관련 키워드가 포함된 기사만 통과시킨다.
    이 단계에서 주제와 무관한 기사들이 제거되므로,
    AI 는 더 좁고 명확한 후보군 안에서 중요한 뉴스를 선정할 수 있다. 
    
**9. Aggregate News**

    필터링을 거친 기사 목록을 AI 가 읽기 쉬운 하나의 텍스트 묶음으로 변환하는 단계이다.
    각 기사별 제목, 링크, 언론사, 발행일시, 설명 등을 한데 모아 AI 입력용 기사 목록을 만든다. 
    이 과정이 있어야 AI 가 여러 개의 기사를 한 번에 비교하고, 
    그중 중요한 뉴스 3 건을 선택할 수 있다. 
    
**10. Top3 선정**

    AI 가 기사 목록에서 오늘 가장 중요한 뉴스 3 건을 선정하는 단계이다. 
    선정 기준은 국민 생활 영향, 경제 영향, 국제 영향, 장기 파급력, 중복 제거, 정치적 편향 방지이다.
    단순히 최신 기사만 고르는 것이 아니라, 사회적·경제적 파급력이 큰 기사를 우선적으로 선택하도록 설계했다. 
    
**11. Parse Top3**

    AI 가 반환한 Top3 선정 결과를 다시 구조화된 데이터로 변환하는 단계이다. 
    AI 응답은 텍스트 형태로 나오기 때문에, 이를 제목, 링크, 언론사, 발행일시, 선정 이유 등의 개별 필드로 분리한다. 
    이 과정을 거쳐 이후 요약 노드와 노션 저장 노드가 각 값을 정확히 사용할 수 있다. 

**12. Check Existing News**

    Notion 데이터베이스에 동일한 뉴스가 이미 저장되어 있는지 확인하는 단계이다.
    Save to Notion 전에 기존 데이터베이스를 조회하여, 새로 저장하려는 기사와 기존 기사 데이터를 비교한다. 
    이 단계에서는 원문 링크를 기준으로 중복 여부를 판단하며, 동일한 링크가 이미 존재할 경우 중복 기사로 분류한다.
    이를 통해 같은 기사가 반복 실행 때마다 중복 저장되는 문제를 방지한다.

**13. Check Duplicate Code**

    Notion에서 가져온 기존 기사 목록과 새로 저장하려는 기사 목록을 비교하여 isDuplicate 값을 생성하는 단계이다.
    기존 데이터베이스에 동일한 원문 링크가 있으면 isDuplicate = true , 없으면 isDuplicate = false 로 표시한다.
    Notion 노드에서 직접 필터링이 어려운 경우를 보완하기 위해, Code 노드에서 중복 여부를 직접 판단하도록 구성했다.

**14. IF**

    Check Duplicate Code에서 생성한 isDuplicate 값을 기준으로 저장 여부를 분기하는 단계이다.
    isDuplicate = true 인 경우 이미 저장된 기사이므로 저장하지 않고 종료한다.
    isDuplicate = false 인 경우 신규 기사로 판단하여 Save to Notion 단계로 전달한다.
    이를 통해 자동화 실행 시 동일 기사중복 저장을 방지하고, Notion 데이터베이스의 품질을 유지한다.

**15. AI 요약**

    선정된 기사 3 건에 대해 각각 짧은 요약문을 생성하는 단계이다. 
    사용자가 원문을 바로 읽지 않아도 핵심 내용을 빠르게 파악할 수 있도록 기사 내용을 간결하게 정리한다.
    요약문은 노션 데이터베이스의 “요약문” 속성에 저장되어 최종 뉴스 콘텐츠로 활용된다. 

**16. Merge Summary**

    Top3 기사 정보와 AI 요약 결과를 하나로 합치는 단계이다. 
    Top3 선정 단계에서 나온 제목, 링크, 언론사, 발행일시 정보와 AI 요약 단계에서 생성된 요약문을 같은 기사 단위로 결합한다. 
    이를 통해 노션에 저장할 최종 데이터 세트가 완성된다. 

**17. Prepare For Notion**

    노션 데이터베이스에 저장하기 적합한 형태로 데이터를 정리하는 단계이다. 
    제목, 요약문, 원문 링크, 발행일시, 언론사, 중복키 등의 값을 노션 속성에 맞게 매핑한다. 
    특히 원문 링크를 기반으로 중복키를 생성해 동일 기사 여부를 확인할 수 있도록 구성했다. 
    
**18. Save to Notion**

    최종 정리된 뉴스 데이터를 노션 데이터베이스에 저장하는 단계이다. 
    저장되는 항목은 제목, 요약문, 원문 링크, 발행일시, 언론사, 중복키 등이다. 
    이를 통해 매일 오전 9 시에 자동으로 수집·선별·요약된 뉴스 콘텐츠가 노션 페이지에 누적된다.
