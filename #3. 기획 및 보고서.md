## 06.24 (수요일)

    1:45 pm 김수아 _ 중복 저장 방지 키 설정, 트리거 설정, 강제 크롤링 방지 관련 의견 제시
    
    -> 중복 저장 방지 키 설정은 RSS 수집 후 유입 과정에서 수집한 기사의 제목과
       데이터베이스(노션)에 저장되어있는 데이터의 중복 단어를 분석해서 중복사항이 있는지
       확인하고 중복사항이 없을 경우에 AI툴 활용해서 자동으로 저장되도록 기능 추가하는거 가능한가요?
       
       트리거 설정 문제는 에러 변수를 최소화해야하니까 기사수집 노드에서 적용하면 어떨까요?
       
       강제 크롤링 방지관련 문제는 미션에 시스템 자체를 아예 자동화로 해야하고
       사람개입이 있으면 안된다는 제약이 있어서 로컬호스트로 해야할 것 같습니다

---

    4:19 pm 김태경 _ 중복 저장 방지 키 설정, 트리거 설정, 강제 크롤링 방지 관련 의견 제시

    -> 동일 기사 중복 저장 방지 키 설정 (키 1개 이상)의 경우 이미 2단계로 중복 방지가 구성되어 있으니,
       방법 3: 현재 Deduplicate News Code 노드 개선 (현재 가장 강력)으로 진행하면 어떨까요?
       거기에다가 수아님이 제안해 주신 부분을 적용하여,
       Save to Notion 바로 앞에 중복 여부 확인 노드를 추가하여
       노션 데이터베이스에 중복기사를 걸러내어 저장하면 조금더 강력한 조합이 되지 않을까 합니다.,
       
       무한루프에 빠지지 않도록 타임아웃은 워크플로우 전체에 적용 하는 것이 좋을 것 같습니다.
       rss 수집 실패시는 계속 진행 하면 어떨까요?
       이미 언론사를 세곳에서 기사를 가져오니 한곳에서 수집 실패가 있더라도
       2곳에서 기사를 가져 올 수 있을 것 같아요.
       ai top 3기사 선정과 기사 요약은 전체 워크플로우의 핵심이니 실패시 퀄리티의 문제가 생길 수 있습니다.
       워크플로우를 중단하고 2차례 정도 재시도를 하여 일시적 오류나 네트워크 문제 발생 시
       복구가 가능할 것 같습니다.,
       
       저작권 문제는 이 워크플로우를 로컬호스트로 (제목만 가져오기, 간단함) 해야겠네요.
